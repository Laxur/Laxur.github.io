Paradigmas de programacion

  • Programacion orientada a objetos
  • Programacion Imperativa
  • Programacion Funcional
  • Programacion Declarativa
  • Programacion Estructural

Programacion orientada a objetos

Se basa en el concepto de "objetos", que pueden contener datos y código. Los programas se estructuran en torno a la interacción entre estos objetos. Ejemplo: Lenguajes como Java, C++, Python y Ruby son orientados a objetos.

Programación Imperativa

En este paradigma, un programa se expresa como una secuencia de declaraciones que cambian el estado de un programa. Las instrucciones indican cómo realizar ciertas operaciones. Ejemplo: Lenguajes como C, Fortran y Pascal son imperativos.

Programación Funcional

Enfocada en funciones matemáticas y evaluación de expresiones. No hay estado mutable, y las funciones son tratadas como ciudadanos de primera clase. Ejemplo: Lenguajes como Haskell, Lisp, y parte de JavaScript (con funciones de orden superior).

Programación Declarativa

Se centra en la especificación de lo que se quiere lograr, sin describir cómo lograrlo. Los programas se escriben en un nivel más abstracto. Ejemplo: SQL es un lenguaje declarativo para manejar bases de datos.

Programacion Estructural

Descripción: Utiliza estructuras de control de flujo como bucles y condicionales para organizar el código en bloques más manejables. Ejemplo: Lenguajes como C y Pascal favorecen la programación estructurada.

  • Lenguajes Imperativos: El lenguaje imperativo es un paradigma de programación que se enfoca en describir cómo se debe realizar una tarea a través de una serie de instrucciones. Ejemplos de lenguajes imperativos son C, Java y Python.

  • Lenguajes Declarativos: El lenguaje declarativo es un paradigma que se enfoca en describir qué se quiere hacer en lugar de cómo hacerlo. Ejemplos de lenguajes declarativos son SQL, Prolog y HTML.

  • El centro del paradigma funcional son las funciones, las cuales son elementos de primer orden y se van a tener funciones de alto orden. Con las funciones puedo asignar referencias de funciones a objetos, puedo pasar una función como parámetro y devolver una referencia a una función

  • Funciones puras, con los mismo valores de entrada produce las mismas salidas ( no se va a meter otra función y alterar el comportamiento de esta ), no se va a tener que modificar variables globales que toquen funciones (Una función pura no modifica, si no que copia el dato)
